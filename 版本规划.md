# 版本规划

## v1.0

- [x] /页面作为首页，同时也是dashboard页面，展示各功能模块的入口、概览、主要指标。
- [x] 所有页面的UI采用统一的设计风格，包括颜色、字体、布局等。采用上中下布局，顶部为导航栏，中间为主要内容区域，底部为操作栏。中间部分有面包屑导航，用于展示当前页面的位置。页面风格清新，有科技感。
- [x] 所有页面除了登录页和注册页，都需要进行认证。未认证时自动跳转登录页，登录后自动返回。
- [x] 应该使用最新版本的next.js 16.1 和antd 6.2 以上版本
- [x] /dashboard 页面左边的导航栏不需要，所有模块链接都在dashboard上展示。

## v1.1

- [ ] /data-source 是外部数据源模块。用于管理系统需要对接的外部数据源，可以是mysql、sql server等关系数据库。下面的子页面包括：
    - [ ] /data-source 数据源的dashboard，以卡片方式展示所有已配置的数据源。
    - [ ] /data-source/create 数据源创建页面，用于添加新的外部数据源。
    
## v1.2

- [ ] 数据元是数据治理体系中用于定义和描述特定业务概念的最小、不可再分的数据单元，其核心在于通过一组标准化的属性实现全组织范围内的统一理解与管理。一个规范的数据元必须包含明确的标识与名称、无歧义的业务定义、具体的表示格式与数据类型、清晰的值域范围、与其他数据标准的关系，以及版本、来源与责任方等全生命周期管理信息，从而确保数据在业务和系统间能够被准确、一致地交换与解释。根据以上的定义修正页面上、需求文档、设计文档中关于数据元的相关内容。
- [ ] /data-elements作为数据元管理页面，下面的子页面包括：
    - [ ] /data-elements 数据元列表，可根据数据元的属性进行
    - [ ] /data-elements/:id 数据元详情
    - [ ] /data-elements/setting 数据元设置页面。在此可以设置数据元的数据源（由“外部数据源模块”中选取）、读取数据元的sql语句、数据元相关数据的同步周期等。
- [ ] dashboard上有数据元的入口等相关信息，且链接地址正确。
